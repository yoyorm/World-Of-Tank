# World-Of-Tank

My first Unity Learning demo

基于你的需求（UI 界面、坦克移动、障碍物、子弹、简单敌人），我建议按以下 5 个阶段推进：

---

### 🎯 **阶段一：基础框架搭建（1-2 天）**

_目标：让游戏"跑起来"_

1. **项目设置**

   - 创建 2D/3D 项目（建议 3D，俯视角）
   - 导入素材：坦克模型、地形、爆炸特效（可去 Unity Asset Store 搜免费资源）
   - **防坑**：提前规划好文件夹结构（Scenes, Scripts, Prefabs, Materials, Audio 等）

2. **基础场景**

   - 创建平面作为战场
   - 设置主摄像机（正交投影 Orthographic，俯视 45 度角）
   - 添加方向光和环境光

3. **玩家坦克**
   - 创建坦克 Prefab，添加 Rigidbody 和 Collider
   - 实现 WASD 移动 + 鼠标/方向键旋转炮塔
   - **防坑**：用`FixedUpdate()`处理物理移动，避免卡顿

---

### 🎯 **阶段二：核心战斗机制（2-3 天）**

_目标：让游戏"好玩起来"_

4. **子弹系统**

   - 创建子弹 Prefab（加刚体，设置速度）
   - 实现空格键发射，限制射速（协程+冷却时间）
   - 子弹碰撞检测，击中后销毁
   - **防坑**：用对象池（Object Pooling）管理子弹，避免频繁 Instantiate 导致性能问题

5. **障碍物系统**

   - 创建三种障碍物：可摧毁的墙、不可摧毁的墙、障碍物
   - 添加 Tag 区分类型（"Destructible", "Indestructible"）
   - 可摧毁墙添加血量，受击后播放爆炸动画并销毁

6. **敌人 AI（简单版）**
   - 创建敌人坦克 Prefab
   - 简单行为：随机移动 + 朝玩家方向射击
   - **AI 逻辑**：用 Vector3.Distance 检测玩家距离，<10 米时追击+开火

---

### 🎯 **阶段三：游戏逻辑完善（1-2 天）**

_目标：让游戏"完整起来"_

7. **胜负判定**

   - 玩家血条（3 条命），敌人子弹击中扣血
   - 消灭所有敌人 → 胜利界面
   - 玩家血量归零 → 失败界面

8. **UI 界面**

   - 开始菜单（Start, Settings, Quit）
   - 游戏内 HUD（玩家血量、剩余敌人数）
   - 胜利/失败界面 + 重试按钮
   - **防坑**：用 Unity 的 Canvas 系统，注意分辨率适配（Canvas Scaler）

9. **音效和特效**
   - 射击音效、爆炸音效、背景音乐
   - 爆炸粒子特效（可用 Unity 自带 Particle System）

---

### 🎯 **阶段四：进阶优化（可选，2-3 天）**

_目标：让游戏"精致起来"_

10. **敌人 AI 进阶**

    - 巡逻状态 → 追击状态 → 攻击状态（有限状态机 FSM）
    - 躲避玩家子弹（简单射线检测）

11. **道具系统**

    - 随机生成：加速、护盾、双发子弹
    - 用触发器（Trigger）拾取

12. **关卡系统**
    - 多波次敌人刷新
    - 难度递增（敌人数量、速度、血量）

---

### 🎯 **阶段五：打磨发布（1 天）**

_目标：让游戏"专业起来"_

13. **性能优化**

    - 对象池扩展应用到敌人和特效
    - 批量静态障碍物勾选 Static，烘焙光照

14. **打包测试**
    - Build Settings 配置，PC 端打包测试
    - 调试日志清理

---

### 🔥 **新手最容易踩的坑（重点 ⚠️）**

| 问题              | 解决方案                                    |
| ----------------- | ------------------------------------------- |
| **物理系统卡顿**  | 移动代码放`FixedUpdate()`，别放`Update()`   |
| **子弹穿墙/穿敌** | 子弹速度别太快，刚体设为 Continuous Dynamic |
| **敌人扎堆**      | 给敌人添加 NavMeshAgent 或简单避障逻辑      |
| **射速过快**      | 用`Time.time`记录上次射击时间，加冷却       |
| **场景混乱**      | 提前规划 Tag 和 Layer，别用 Default         |
| **内存泄漏**      | 频繁生成的对象用对象池，别直接 Destroy      |
| **代码冗余**      | 玩家和敌人坦克继承同一个基类`TankBase`      |

---

### 💡 **建议的开发顺序（从易到难）**

1. 玩家移动 → 2. 发射子弹 → 3. 子弹碰撞 → 4. 创建敌人 → 5. 敌人简单 AI → 6. 胜负判定 → 7. UI → 8. 音效 → 9. 优化

这样每完成一步都能看到效果，保持学习动力！
